- **Service Orchestration**: Uses `docker-compose.yml` to define and link five distinct services: `postgres` (data), `ollama` (LLM inference), `backend` (core API), `frontend` (SPA), and a voice screening triad (`livekit`, `speech-service`, `voice-agent`).
- **API Gateway & Routing**: Nginx acts as the single entry point, routing static SPA assets, proxying `/api/*` requests to the FastAPI backend, and forwarding `/livekit` WebSocket traffic to the LiveKit SFU for real-time voice interactions.
- **Shared State & Persistence**: All services share a centralized PostgreSQL database for tenant data, while Ollama volumes persist LLM models. The backend manages shared infrastructure concerns like JWT authentication, rate limiting, and background job queues.
- **Inter-Service Communication**: The backend communicates with the voice agent via HTTP dispatch; the voice agent coordinates with LiveKit for WebRTC sessions and the speech service for STT/TTS, creating a decoupled voice screening pipeline.
